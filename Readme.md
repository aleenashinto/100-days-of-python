HackerRank Challenge: Maximize It!
-------------------------------------
Problem Statement
-----------------------
You are given a function f(X)=X2. You are also given K lists. The list consists of N, elements.
You have to pick one element from each list so that the value from the equation below is maximized:
S(f(X)+f(X2)+...+f(X))%M
X, denotes the element picked from the list. Find the maximized value S obtained.
% denotes the modulo operator.
Note that you need to take exactly one element from each list, not necessarily the largest element. You add the squares of the chosen elements and perform the modulo operation. The maximum value that you can obtain, will be the answer to the problem.
Input Format
The first line contains 2 space separated integers K and M.
The next K lines each contains an integer N,, denoting the number of elements in the list, followed by N, space separated integers denoting the elements in the list.
Constraints
1<K≤7
1 ≤ M < 1000
1≤ N ≤7
1 Magnitude of elements in list < 10
Output Format
Output a single integer denoting the value S
Sample Input
3 1000
254
3789
5 5 7 8 9 10
Sample Output
206
Explanation
Picking 5 from the 1 list, 19 from the 2nd list and 10 from the 3rd list gives the maximum & value equal to (52+92+102)%1000 = 206.


Example
Input
3 1000
2 5 4
3 7 8 9
5 5 7 8 9 10
Output
206
Explanation

Choose:

5 from List 1
9 from List 2
10 from List 3

Then:

(5² + 9² + 10²) % 1000
= (25 + 81 + 100) % 1000
= 206
Approach

Since:

1 ≤ K ≤ 7
1 ≤ Nᵢ ≤ 7

The total number of combinations is relatively small:

7⁷ = 823543

This means we can generate every possible combination and compute the result efficiently.

Algorithm
Read all lists.
Generate all possible combinations using itertools.product().
For each combination:
Calculate the sum of squares.
Take modulo M.
Update the maximum value.
Print the answer.
Python Solution
from itertools import product

K, M = map(int, input().split())

lists = []

for _ in range(K):
    arr = list(map(int, input().split()))
    lists.append(arr[1:])

answer = 0

for combination in product(*lists):
    value = sum(x * x for x in combination) % M
    answer = max(answer, value)

print(answer)
Complexity Analysis
Time Complexity
O(N₁ × N₂ × ... × Nₖ)

Worst case:

O(7⁷)

which is approximately:

823543 operations

This easily fits within the problem constraints.

Space Complexity
O(K)

excluding the combinations generated by itertools.product().
